# 量能与换手确认 V1 设计

## 背景

当前系统已经具备缠论结构、中枢、买卖点、市场环境、技术辅助、买入否决和回测验证能力。下一步需要把成交量纳入判断，让平台能识别“上涨有没有量支持”“回调是不是缩量”“下跌是不是放量风险”。

现有日 K 线数据包含 `volume` 和 `amount`，适合做历史成交量与成交额分析。但当前日线数据源没有稳定的历史换手率字段，换手率只能从行情快照或其他数据源补充。因此第一版以成交量和成交额为主，换手率作为可用则展示、不可用则不参与强判断的辅助信息。

## 目标

1. 新增 `VolumeContext`，总结成交量、成交额和可用换手率。
2. 将量能确认接入信号评分：对买入信号加分、减分或提示风险。
3. 将关键量能风险接入买入否决：避免无量上涨、放量下跌时误买。
4. Web、CLI JSON、CSV 列表展示量能状态。
5. 回测自然使用接入后的信号，后续可验证量能条件是否提升信号质量。

## 非目标

本阶段不做复杂筹码分布，不做历史换手率回填，不引入分时成交明细，不用单一换手率阈值直接决定买卖。换手率数据不可用时不阻塞分析，也不把成交量强行换算成换手率。

## 数据模型

新增 `VolumeContext`：

- `label`: `助力`、`拖累`、`蓄势`、`中性`。
- `score`: 0 到 1 的辅助分。
- `volume_label`: 成交量状态，例如 `放量上涨`、`缩量回调`、`放量下跌`、`无量上涨`、`量能平稳`。
- `volume_ratio_5`: 最新成交量相对 5 日均量的倍数。
- `volume_ratio_20`: 最新成交量相对 20 日均量的倍数。
- `amount_ratio_5`: 最新成交额相对 5 日均额的倍数。
- `turnover_pct`: 当前换手率，可为空。
- `turnover_label`: 换手状态，例如 `换手活跃`、`换手温和`、`换手不足`、`数据不足`。
- `summary`: 面向用户的一句话说明。
- `reasons`: 量能判断依据。
- `risk_notes`: 量能风险提示。

`ChanSignal` 增加 `volume_context: VolumeContext | None`。

## 成交量规则

第一版使用最近 20 个交易日的成交量和成交额做相对比较：

1. 放量上涨：最新收盘上涨，成交量大于 5 日均量的 1.3 倍，视为买入助力。
2. 缩量回调：最新收盘下跌，但成交量低于 5 日均量的 0.85 倍，视为回调抛压不强。
3. 放量下跌：最新收盘下跌，成交量大于 5 日均量的 1.3 倍，视为风险拖累。
4. 无量上涨：最新收盘上涨，但成交量低于 5 日均量的 0.85 倍，视为买入信号需要打折。
5. 量能平稳：其余场景，作为中性信息展示。

成交额用于辅助确认成交量结论。若量增但成交额没有同步改善，提示“量价配合一般”；若成交额同步放大，说明资金参与更明显。

## 换手率规则

第一版只接收可用的当前换手率：

1. 换手率为空：显示 `数据不足`，不参与评分。
2. 换手率低于 1%：显示 `换手不足`，买入信号轻微减分。
3. 换手率 1% 到 5%：显示 `换手温和`。
4. 换手率高于 5%：显示 `换手活跃`，若同时放量上涨则增强助力；若同时放量下跌则增强风险。

换手率不作为独立买卖依据，只和成交量、价格方向组合使用。

## 信号接入

`ChanAnalyzer.analyze()` 生成日线数据后构建 `VolumeContext`，并传入 `ChanSignalEngine.evaluate()`。

买入信号：

- `放量上涨`：提高信号扎实程度。
- `缩量回调`：三买、二买类回踩信号加分。
- `无量上涨`：降低信号扎实程度，并加入风险提示。
- `放量下跌`：触发买入否决或强风险提示。

卖出信号：

- `放量下跌`：增强卖出/减仓信号。
- `放量上涨` 但存在上行背驰风险：提示高位放量分歧，需要谨慎追高。

观察信号：

- 展示量能状态，不强行改变动作。

## 买入否决接入

新增量能否决条件：

1. 买入类信号遇到 `放量下跌`，直接否决买入。
2. 三买候选回踩时，如果价格跌回中枢且成交量放大，直接否决买入。
3. 无量上涨叠加 30 分钟确认缺失，组合否决买入。

否决文案示例：

- “虽然出现买点，但最新一天下跌同时成交量明显放大，说明抛压增强，买入暂缓。”
- “价格回到中枢内且回落放量，三买确认失败，先观察。”

## 展示方式

Web 单股面板新增“量能换手”：

- 主标签显示 `助力`、`拖累`、`蓄势`、`中性`。
- 说明文本显示成交量倍数、成交额变化和换手率状态。

CSV 批量结果新增“量能”列，便于快速扫描。

JSON 输出新增 `volume_context`，供后续回测和外部分析使用。

## 测试计划

1. `tests/test_volume_context.py`
   - 放量上涨识别为助力。
   - 缩量回调识别为蓄势或助力。
   - 放量下跌识别为拖累并给出风险。
   - 无量上涨识别为拖累或中性偏弱。
   - 数据不足返回中性可读说明。
2. `tests/test_signals.py`
   - 量能助力提高买入信号理由。
   - 放量下跌触发买入否决。
   - 非买入信号不被量能强行改写。
3. `tests/test_chan_cli.py` 和 `tests/test_web.py`
   - CLI 表格和 Web 页面出现量能字段。
4. 全量测试和 CLI 冒烟，确认旧流程不破坏。

## 成功标准

1. 单股分析能读到量能状态和白话解释。
2. 买入信号遇到放量下跌或无量上涨等风险时明显降级。
3. 换手率不可用时系统仍可运行，且明确显示数据不足。
4. JSON、Web、CSV 批量列表都包含量能信息。
5. 测试通过，且本阶段变更可独立回退。
