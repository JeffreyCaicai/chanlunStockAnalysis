# 标准缠论结构加强 V1 设计

## 背景

当前系统已经具备日线分型、笔、趋势、背驰风险、买卖点、市场环境、技术辅助和回测验证能力。现有缠论结构仍偏简化：三买、三卖主要依赖趋势和最近一笔方向判断，缺少标准缠论里更关键的“中枢”结构。

这一阶段目标是把“中枢”作为新的结构层加入系统，让买卖点判断更贴近实战缠论，同时保持现有功能稳定可回退。

## 目标

1. 在现有分型和笔之上识别最近中枢。
2. 用中枢关系强化三买、三卖判断。
3. 在 Web、CLI JSON 和结构摘要中展示更容易理解的标准结构信息。
4. 保持原有信号逻辑作为回退，避免新版结构过早覆盖全部判断。
5. 增加单元测试，保证中枢和三买三卖规则可验证。

## 非目标

本阶段不做完整缠论重写，不引入递归级别、不实现严格线段破坏算法、不做多周期自动联立推理。30 分钟确认、市场环境、技术辅助、回测能力只做兼容接入，不在这一阶段重构。

## 方案选择

采用保守增强方案：在 `src/astockdata/chan.py` 中新增标准结构数据模型和中枢识别逻辑，在 `src/astockdata/chan_points.py` 中优先使用中枢规则判断三买、三卖。现有一买、二买、一卖、二卖和趋势兜底逻辑继续保留。

相较于完整重写，这个方案风险较低；相较于只改文案或规则，它能真正把缠论的中枢判断能力接入系统。

## 数据模型

新增 `CentralZone`，表示一个由连续三笔重叠形成的中枢：

- `start_timestamp`：中枢开始时间。
- `end_timestamp`：中枢结束时间。
- `low`：中枢下沿。
- `high`：中枢上沿。
- `stroke_count`：参与形成中枢的笔数。
- `direction`：中枢形成后的离开方向，取 `up`、`down`、`inside`。

`ChanStructure` 增加 `zones: list[CentralZone]` 字段。为了降低影响，旧字段保持不变，信号、回测和 UI 可以逐步消费新字段。

## 中枢识别规则

第一版使用连续三笔的价格区间重叠来识别中枢：

1. 每一笔的价格区间为起点价和终点价之间的范围。
2. 连续三笔若存在共同重叠区间，则形成一个中枢。
3. 中枢上沿取三笔区间高点中的最小值。
4. 中枢下沿取三笔区间低点中的最大值。
5. 若后续相邻笔仍与中枢有重叠，则扩展中枢结束时间和参与笔数。
6. 最新价格或最近一笔完全离开中枢上方，标记为 `up`；完全离开中枢下方，标记为 `down`；仍在中枢内标记为 `inside`。

这不是完整线段级别中枢，但它比现有纯趋势判断更接近标准缠论结构，适合作为第一版。

## 买卖点增强

优先强化三买和三卖：

- 三买：价格向上离开最近中枢后，最近向下回踩不跌回中枢下沿，且回踩低点位于中枢上沿附近或上方。
- 三卖：价格向下离开最近中枢后，最近向上反抽不重新进入中枢上沿，且反抽高点位于中枢下沿附近或下方。
- 如果中枢条件不足，继续使用当前简化三买、三卖规则。

信号解释需要直接说明结构关系，例如：

- “价格已经离开最近中枢上方，回踩没有跌回中枢，属于三买候选。”
- “价格已经离开最近中枢下方，反抽没有回到中枢，属于三卖风险。”

## 结构摘要展示

`StructureSummary` 增加最近中枢信息，Web 结构状态展示：

- 最近中枢：时间范围和上下沿。
- 当前位置：中枢上方、中枢内部、中枢下方。
- 结构含义：例如“离开中枢上方后回踩确认，偏三买观察”。

展示口径保持通俗，不把用户逼进术语堆里。

## 回测兼容

回测不新增入口参数。新版 `classify_trade_point` 产生的三买、三卖信号会自然进入现有回测统计，后续可以通过回测结果观察中枢规则是否改善胜率、平均收益和回撤。

## 测试计划

1. `tests/test_chan.py` 增加中枢识别测试：
   - 三笔重叠形成中枢。
   - 不重叠时不形成中枢。
   - 后续笔重叠时扩展中枢。
   - 离开方向可识别。
2. `tests/test_chan_points.py` 增加买卖点测试：
   - 中枢向上离开后回踩不回中枢，识别三买。
   - 中枢向下离开后反抽不回中枢，识别三卖。
   - 中枢不足时继续走旧规则。
3. `tests/test_signals.py` 增加结构摘要字段测试。
4. 全量运行现有测试，确保旧能力不回退。

## 成功标准

1. 分析结果能看到最近中枢的上下沿和位置关系。
2. 三买、三卖说明从“趋势回踩/反弹”升级为“中枢离开后的回踩/反抽确认”。
3. CSV、单股分析、CLI JSON、回测流程不破坏。
4. 单测通过，且变更可以和上一版本清晰区分。
