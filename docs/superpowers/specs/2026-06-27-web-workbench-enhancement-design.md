# Web 实战工作台增强 V1 设计

## 背景

当前 Web 页面已经能输入股票代码或名称、导入 CSV、查看结构状态、交易信号、K 线走势、量能、否决条件和 JSON。随着缠论结构、回测、否决条件、量能换手陆续接入，页面信息变多，但信息层级仍偏“堆叠”：用户需要在三栏之间来回找重点。

第五优先级采用 C 版混合实战模式，把页面增强为本地“扫描 + 研判”工作台。目标是让用户打开页面后先看到交易结论，再看为什么，最后决定是否继续观察或加入候选。

## 目标

1. 重排 Web 页面为更清晰的实战工作台：左侧操作，中间单股研判，右侧 CSV 扫描结果。
2. 增加顶部关键结论卡，优先显示动作、结构、买卖点、量能、否决条件、下一步观察价位。
3. 增强 CSV 批量结果：提供动作、否决状态、量能状态筛选，并让点击行联动详情。
4. 增强 K 线图：在现有 SVG 图上标注最近顶分型、底分型、中枢上下沿、买卖点参考价和失效价。
5. 将原因、风险、失效条件、JSON 分层展示，降低一次性阅读压力。

## 非目标

本阶段不修改核心交易信号算法，不新增后端 API，不引入图表库，不做登录、多用户、云端部署、交易账户或自动下单。CSV 仍然只需要 `code` 一列，已有可选 `cost`、`position` 字段继续兼容。

## 页面结构

V1 保持单页本地 HTML，不拆前端构建系统。页面从当前三栏重排为：

- 顶部：服务状态和当前股票身份。
- 左侧操作区：单股输入、运行分析、CSV 上传、CSV 示例、JSON 复制。
- 中间研判区：关键结论卡、K 线图、结构摘要、解释分层。
- 右侧扫描区：CSV 批量汇总、筛选控件、批量结果表。

小屏幕下仍然单列堆叠，顺序为操作区、研判区、扫描区。

## 关键结论卡

新增 `decisionStrip` 区域，包含 6 个信息卡：

1. 交易动作：显示 `买入`、`卖出`、`继续持有`。
2. 结构方向：显示日线趋势和中枢位置。
3. 买卖点：显示一买、二买、三买、一卖等标签。
4. 量能换手：显示放量上涨、缩量回调、放量下跌、无量上涨等状态。
5. 否决条件：显示未触发或已否决买入。
6. 下一步观察：从失效条件、最近中枢或买卖点中提取最重要的一句话。

这些卡片只做展示和阅读排序，不改变信号计算。

## CSV 扫描增强

CSV 批量结果保留现有表格，新增筛选控件：

- 动作筛选：全部、买入、卖出、继续持有。
- 否决筛选：全部、只看已否决、排除已否决。
- 量能筛选：全部、助力、蓄势、拖累。

筛选在浏览器本地完成。`renderPortfolio(results)` 保存完整结果到前端状态，筛选变化后重新渲染表格。点击任一行仍调用 `renderSignal(item)` 刷新单股详情。

## K 线图增强

继续使用当前 SVG 实现，不引入第三方图表库。`renderKlineChart(signal)` 从完整信号读取：

- `recent_klines`: 绘制最近日 K 线。
- `daily_summary.latest_top`: 标注最近顶分型。
- `daily_summary.latest_bottom`: 标注最近底分型。
- `daily_summary.latest_zone`: 标注中枢上下沿。
- `trade_point`: 标注买卖点参考价。
- `invalidations[0]`: 尝试提取第一个失效价并绘制失效线。

如果某类数据不存在，就跳过对应标注，页面不能报错。

## 解释分层

当前页面已有原因、失效条件、风险提示、背驰说明和 JSON。V1 改成分层展示：

- “关键原因”：显示 `reasons`。
- “风险提示”：显示 `risk_notes`。
- “失效条件”：显示 `invalidations`。
- “原始 JSON”：保留复制和查看完整数据。

风险提示不再只放在右侧一行，避免重要风险被压缩成一段难读文字。

## 前端状态

新增浏览器端状态：

- `latest`: 当前选中的单股信号或批量结果。
- `portfolioResults`: 当前 CSV 批量完整结果。
- `portfolioFilters`: 当前筛选条件。

新增前端辅助函数：

- `structureDirectionText(signal)`
- `nextWatchText(signal)`
- `invalidPrice(signal)`
- `renderDecisionStrip(signal)`
- `filterPortfolioResults(results)`
- `setActivePortfolioRow(row)`
- `renderKlineChart(signal)`

## 错误处理

- 单股分析失败仍使用现有 `alert`。
- 未选择 CSV 文件仍提示“请选择 CSV 文件”。
- CSV 结果为空时显示“尚未导入 CSV”或“没有符合筛选条件的股票”。
- K 线标注数据缺失时跳过，不影响 K 线基础绘制。
- 复制 JSON 失败时保留“复制失败，请手动复制 JSON”。

## 测试计划

1. `tests/test_web.py`
   - HTML 包含工作台关键区域：`decisionStrip`、CSV 筛选控件、风险列表、K 线图例。
   - HTML 包含新增前端函数：`renderDecisionStrip`、`filterPortfolioResults`、`nextWatchText`、`invalidPrice`。
   - 页面仍包含原有接口路径、CSV 示例、K 线图、复制失败文案。
2. 全量 `unittest` 确认 Web HTML 改造不破坏已有 API。
3. 浏览器验证：
   - 本地 Web 服务能打开。
   - 首页不是空白，没有浏览器控制台错误。
   - 输入股票代码运行分析后，关键结论卡、K 线图、解释区更新。
   - 桌面和移动视口无明显重叠或截断。

## 成功标准

1. 用户打开页面后，第一眼能看到交易动作和关键判断依据。
2. CSV 批量扫描可以按动作、否决和量能快速筛选。
3. 点击 CSV 结果行能刷新单股详情。
4. K 线图能显示结构参考线，帮助快速定位最近顶底、中枢和失效价。
5. 页面仍是本地单页 Web UI，测试通过，改动可独立回退。
